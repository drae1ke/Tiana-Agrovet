
# Agrovet Management System - Implementation Plan

## Overview
A comprehensive Point of Sale (POS) and inventory management system designed specifically for agrovet businesses (agricultural and veterinary supplies). The system features bilingual support (English/Kiswahili), smart inventory management with auto-notifications and auto-ordering, M-Pesa payment integration, and expiry date tracking.

---

## Core Features

### 1. Bilingual Support (English/Kiswahili)
- Language toggle in the header for switching between languages
- All UI text, labels, and messages available in both languages
- Examples:
  - Dashboard / Dashibodi
  - Stock Levels / Viwango vya Bidhaa
  - Low Stock Alert / Tahadhari ya Bidhaa Chache
  - Add Product / Ongeza Bidhaa

### 2. Admin Authentication
- Simple admin login using localStorage
- Default credentials stored securely
- Session management with auto-logout

### 3. Dashboard
- Overview cards showing:
  - Total Products / Jumla ya Bidhaa
  - Low Stock Items / Bidhaa Chache
  - Expiring Soon / Zinazoisha Hivi Karibuni
  - Today's Sales / Mauzo ya Leo
- Visual charts for sales trends
- Quick action buttons

### 4. Point of Sale (POS)
- Product search with barcode/name
- Cart management
- Multiple payment methods (Cash, M-Pesa)
- Receipt generation
- Customer phone number capture for M-Pesa

### 5. Inventory Management
- Product CRUD operations
- Categories (Seeds, Fertilizers, Pesticides, Veterinary, Tools)
- Stock level tracking
- Batch/lot number tracking
- Expiry date management

### 6. Auto-Notifications
- Low stock alerts (configurable threshold)
- Expiry date warnings (30, 14, 7 days before)
- Toast notifications and dashboard alerts

### 7. Auto-Order System
- Trusted supplier management
- Automatic purchase order generation for low stock items
- Order history tracking
- Supplier contact integration

### 8. M-Pesa Integration
- STK Push simulation (UI mockup for demo)
- Payment confirmation workflow
- Transaction history

### 9. Expiry Date Tracking
- Visual indicators (color-coded badges)
- Expiry reports
- FIFO (First In, First Out) recommendations

## Technical Details

### localStorage Structure
```text
agrovet_admin - Admin credentials hash
agrovet_session - Current session token
agrovet_products - Products array
agrovet_suppliers - Suppliers array
agrovet_sales - Sales history
agrovet_orders - Purchase orders
agrovet_settings - App settings (language, thresholds)
```

### Auto-Order Logic
1. On product update, check if quantity is less than or equal to minStockLevel
2. Find trusted supplier for the product
3. If autoOrderEnabled for supplier, generate purchase order
4. Show notification to admin about auto-generated order
5. Orders require manual confirmation before sending to supplier

### M-Pesa Integration (Simulation)
- Display M-Pesa payment modal
- Capture customer phone number
- Show STK Push simulation with countdown
- Generate mock transaction reference
- Store payment details with sale record

## Security Considerations
- Admin password hashed before storing
- Session timeout after inactivity
- Input validation using Zod
- No sensitive data exposed in localStorage (only hashed passwords)
